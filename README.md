# AutoNova Booking

AutoNova Booking — это сайт для бронирования автомобилей. Пользователь выбирает машину, указывает даты поездки и отправляет заявку. Сотрудники видят заявки, подтверждают или отменяют бронирования, а инспектор автопарка следит за техническим статусом машин.

Проект сделан на Django. Для обычных страниц используется HTML-интерфейс, а для API используется Django Ninja. API можно открыть отдельно и посмотреть документацию.

## Как объяснить проект на защите

Это система бронирования автомобилей с разными ролями пользователей. На сайте есть каталог машин, регистрация клиентов, единый вход для клиентов и сотрудников, личные кабинеты, управление заявками и проверка доступности автомобиля по датам.

Главная идея: один и тот же автомобиль нельзя забронировать на пересекающиеся даты. Если машина уже занята, сайт покажет статус и не даст создать конфликтующее бронирование.

## Что умеет сайт

- показывает каталог автомобилей с фото, ценой, классом и статусом;
- позволяет клиенту зарегистрироваться и войти в личный кабинет;
- позволяет клиенту создать заявку на бронирование автомобиля;
- показывает клиенту его бронирования;
- запрещает бронировать машину на занятые даты;
- запрещает бронирование на прошедшую дату;
- проверяет возраст клиента и водительский стаж;
- дает менеджеру возможность подтверждать, отменять и завершать заявки;
- дает инспектору возможность менять технический статус автомобиля;
- дает администратору доступ к пользователям, аналитике и Django admin;
- содержит API на Django Ninja с документацией.

## Роли пользователей

| Роль | Что может делать |
|---|---|
| Администратор | Полный доступ: пользователи, заявки, аналитика, автопарк, Django admin |
| Менеджер | Работает с заявками: подтверждает, отменяет, завершает бронирования |
| Клиент | Смотрит каталог, бронирует автомобили, отменяет свои активные заявки |
| Инспектор автопарка | Следит за техническим состоянием автомобилей и меняет их статус |

## Как запустить на Windows

Самый простой способ:

```bat
start_windows.bat
```

После запуска открыть в браузере:

```text
http://127.0.0.1:8000/
```

## Ручной запуск

Если запускать вручную:

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py runserver
```

После этого открыть:

```text
http://127.0.0.1:8000/
```

## Что показать преподавателю

1. Открыть главную страницу.
2. Перейти в каталог автомобилей.
3. Показать карточки автомобилей, фото, цены и статусы.
4. Войти под клиентом `client1`.
5. Открыть автомобиль и попробовать создать бронирование.
6. Показать личный кабинет клиента и список его броней.
7. Войти под менеджером `manager`.
8. Показать раздел заявок, где менеджер может подтвердить или отменить бронь.
9. Войти под инспектором `inspector`.
10. Показать раздел автопарка и изменение технического статуса автомобиля.
11. Войти под администратором `admin`.
12. Показать пользователей, аналитику и Django admin.
13. Открыть `/api/docs` и показать, что есть API на Django Ninja.

## Как показать запрет двойного бронирования

1. Войти под `client1`.
2. Открыть автомобиль, у которого есть занятые интервалы.
3. Попробовать выбрать даты, которые пересекаются с уже существующим бронированием.
4. Сайт покажет ошибку, что автомобиль уже забронирован на выбранные даты.

Это важная часть проекта: система не просто сохраняет заявки, а проверяет правила бронирования.

## Как показать разные права доступа

Можно объяснить так:

- клиент видит каталог и свои бронирования;
- менеджер видит заявки всех клиентов;
- инспектор видит технический статус автомобилей;
- администратор видит все разделы.

Один вход используется для всех: клиент и сотрудник заходят через одну страницу входа, но после входа сайт открывает разные возможности в зависимости от роли.

## Короткий текст для защиты

AutoNova Booking — это система онлайн-бронирования автомобилей. В проекте есть четыре роли: администратор, менеджер, клиент и инспектор автопарка. Проект содержит готовые автомобили, пользователей, бронирования и API-документацию на Django Ninja.
